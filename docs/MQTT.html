AqualinkD integrates with HomeKit, Home Assistant & Domoticz through MQTT. Prerequisite to this section is you have MQTT broker running on your network.
  • The main MQTT server is configured with below options on AqualinkD config
    • mqtt_address host:port example localhost:1883
    • mqtt_user Optional username
    • mqtt_passwd Optional password
    • mqtt_aq_topic Topic for AqualinkD, default=aqualinkd
  • If you want AqualinkD to configure Homeassistant / Domoticz and other hubs through MQTT discovery, then add the below
    • mqtt_discovery_topic the discovery topic, default=homeassistant
    • mqtt_discovery_use_mac In HA (maybe others) you can group entities to one device, this will set that for you using the MAC address
  • Most home automation hubs default to receiving units in metric (ie deg C) and will convert to what ever the user has selected. The below will handle this for you
    • convert_mqtt_temp_to_c = yes REQUIRED for Homekit/bridge/domoticz, highly desirable for HomeAssistant
  • Some Automation hubs will consider a device dead if it doesn't frequently get updates. AqualinkD defaults to "don't send update if nothing has changed", this setting forces AqualinkD to send ALL information every 5 minutes even if nothing has changed
    • mqtt_timed_update = yes REQUIRED for HomeAssistant
  • For all automation hubs, PLEASE make sure you have forced your devices as described in this section
