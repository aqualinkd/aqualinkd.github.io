AqualinkD is set to autoconfigure by default, so unless that didn't work, or you want to manually change ID's, you can skim over this part.

AqualinkD needs a list of device ID's it can use against the Jandy Control Panel. Jandy has introduced many different protocols that over the years to support different interfaces, and some are faster that others at doing particular tasks. Since there is not one "perfect protocol", AqualinkD will use as many as it can (depending on what your panel supports) and will decide on which to use for what tasks appropriately. More details here

These configuration options are below. It is important to note these ID's CAN NOT conflict with the ID's your Jandy keypads/interfaces are using
  • device_id (Mandatory) The main ID of AqualinkD. Set this to 0xFF and AqualinkD will try to auto configure all the below. Valid RS ID's are 0x0a 0x0b 0x09 0x08
  • rssa_device_id(Optional, highly desirable) RSSerialAdapter Valid is is 0x48
  • extended_device_id(Optional) Best ID's are AqualinkTouch (0x30, 0x31, 0x32 & 0x33), if not supported then use OneTouch (0x40, 0x41, 0x42 & 0x43)
  • enable_iaqualink(Optional) If using a AqualinkTouch ID above, and you DO NOT have an iAqualink device (WiFi interface), then set this to YES
