Complete details are in AqualinkD wiki

AqualinkD has many configuration options not listed in these pages, see the wiki (linked above) for complete list. Below are a few of the most common ones.
  • log_level Level of logging valid values are (DEBUG, INFO, NOTICE, WARNING)
  • socket_port Port for web server default=80
  • serial_port Port for RS485 adapter Examples (/dev/ttyUSB0. /dev/ttyS0, /dev/serial0
  • rs485_frame_delay Some Jandy panels are slow (PDA for example), and you need to slow down AqualinkD in how it quickly it reply's to RS485 messages in milliseconds. 20 would be MAX, 0 min.
  • sync_panel_time Keep the Jandy panel time synced with systemtime. Make sure to set systemtime / NTP correctly.
Jandy control panel will turn off the pump sometimes when you don't want it to, ie If the following events happen Freeze protect turned off, Boost turned off, System Powered off/on, during the normal pump on hours, the pump will turn off until the next on cycle. The below options will detect the event override / force the pump on.
  • event_poweron_check_pump = YES System powered on (power outage)
  • event_freezeprotectoff_check_pump = YES Freeze protect turned off
  • event_boostoff_check_pump = YES Boot ended
  • For the above events, you can set the time the pump should be on.
  • event_check_use_scheduler_times Look in AqualinkD scheduler for pump on / pump off times
  • If you don;t use the above, use the below.
  • event_check_pumpon_hour Set a specific on time
  • event_check_pumpoff_hour Set a specific off time
