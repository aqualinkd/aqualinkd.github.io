Reading devices

AqualinkD can read a lot more information from your RS485 devices that Jandy control panel does not show or display, things like
  • Full and correct status of SWG, VSP, Heaters ie "Needs Cleaning" rather than "General Fault"
  • Full RPM, GPM, Watts of all VSP's
  • VSP Pressure curve, Status, Modes
To make use of these please set the appropriate options below depending on your devices.
  • read_RS485_swg = yes for all manufacturers Salt Water Generators
  • read_RS485_ePump = yes for Jandy ePumps
  • read_RS485_vsfPump = yes for all Pentair pumps
  • read_RS485_JXi = yes for Jandy JXi heater
  • read_RS485_LX = yes for Jandy LX heaters
  • read_RS485_Chem = yes for Jandy Chem feeders
  • read_RS485_HeatPump = yes for supported Heatpumps
  • read_RS485_iAqualink = yes If you have Jandy Aqualink WiFi (1,2 or 3)


Forcing devices

AqualinkD will detect certain devices and settings and will configure these devices and provide different options appropriately. But it they are not available at startup, it can create problems for other integrations such as Homekit & Homeassistant.
Example would be a SWG is only on when the filter pump is running, so if the filter pump is off when AqualinkD starts it will not create the appropriate SWG services. This can be annoying for home hub integrations as buttons / options will disappear and re-appear. To fix this, you should force on any devices you have.
  • force_swg = yes for Salt Water Generators
  • force_ps_setpoints = yes for pool or/and spa Heaters
  • force_frzprotect_setpoints = yes If you have freeze protect enabled
  • force_chiller = yes for chillers
